# Retained Modeling Compliance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring `surgeist-retained` into closer compliance with `guidance/surgeist-rust-modeling-guide.md` by tightening public state mutation boundaries, separating canonical and projection replacement semantics, newtyping model revisions, and removing premature public surface.

**Architecture:** Keep retained as the owner of identity, patch atomicity, projection transactions, revision semantics, and event routing. The implementation should preserve the existing public front door from `src/lib.rs`, but make phase distinctions explicit: app-authored state patches are separate from host-owned interaction state, canonical replacement semantics are separate from projection identity replacement semantics, and model revisions become a retained-owned semantic type rather than a raw counter.

**Tech Stack:** Rust 2024 crate, standard library only, existing unit tests in `src/tests.rs`, public API artifact generated by `api/generator`.

---

## Current Modeling Gaps

This plan addresses the review findings against `guidance/surgeist-rust-modeling-guide.md`:

- `StatePatch` exposes public fields for host/input-owned state, so callers can bypass the builder API and mutate states that should remain model- or host-owned.
- Canonical `Patch::Replace` reuses projection `ReplaceMode`, but only `PreserveCompatible` has distinct canonical behavior.
- `Model::revision()` and `Snapshot::revision()` expose raw `u64`, even though the guide calls revisions a strong newtype candidate.
- `StateFlag` is exported from `src/lib.rs` but does not currently participate in an intentional public API.
- There is no direct regression test that failed projection transactions leave `Model::revision()` unchanged.

## Public API Impact

This plan intentionally changes public API shape because the current surface exposes loosely modeled states:

- `StatePatch` stops carrying host/input-owned fields and its remaining fields become private. This is a breaking API tightening; callers must use existing builder methods such as `StatePatch::new().selected(true)`.
- Projection replacement mode moves from `ReplaceMode` to `ProjectionReplaceMode` for `ProjectionEdit::new`. This is a breaking but clarifying phase split; projection callers must use `ProjectionReplaceMode::{PreserveCompatible, PreserveIdentity, ResetIdentity}`.
- Canonical `Patch::Replace` keeps the public name `ReplaceMode`, but its variants become canonical-only: `PreserveCompatible` and `AllowKindChange`. This removes projection-only variants from canonical replacement.
- `Model::revision()` and `Snapshot::revision()` change from `u64` to `ModelRevision`. This is a breaking API tightening; downstream cache adapters can call `.get()` when they need a raw `u64`.
- `StateFlag` is removed from the public front door unless implementation discovers real crate-local usage. This is a breaking removal of premature public surface.
- `State` changes from public fields to `pub(crate)` storage plus public read accessors. This is a breaking API tightening; snapshot consumers should call methods such as `state.selected()` and `state.presence()`.
- `api/public-api.txt` must be regenerated after each public API task so the artifact remains source-derived.

## File Structure

- Modify `src/state.rs`: remove host/input-owned fields from `StatePatch`, make `State` accessor-only at the public boundary, preserve app-mutable builder methods, and remove unused `StateFlag`.
- Modify `src/mutation.rs`: split canonical replacement mode from projection replacement mode by introducing `ReplaceMode` for canonical patches and `ProjectionReplaceMode` for projection edits, or by renaming the current projection-owned mode and adding a canonical-specific mode.
- Modify `src/projection.rs`: move projection-specific replacement behavior to `ProjectionReplaceMode`.
- Modify `src/model.rs`: update match arms and helper signatures to use canonical and projection-specific replacement types; keep transaction/revision behavior unchanged except for adopting the new revision type.
- Modify `src/snapshot.rs`: return the new revision type.
- Modify `src/change.rs` only if the revision newtype needs report-facing helper text or tests expose revision alongside changes. Prefer no change here.
- Modify `src/lib.rs`: export only intentional public types.
- Modify `src/tests.rs`: add focused TDD coverage for each boundary.
- Modify `api/public-api.txt`: refresh with `cargo run --manifest-path api/generator/Cargo.toml` after public API changes.

```rust
/// Monotonic retained model revision for snapshot cache identity.
///
/// The value advances after committed snapshot-observable changes.
#[derive(Clone, Copy, Debug, Default, Eq, Hash, Ord, PartialEq, PartialOrd)]
pub struct ModelRevision(u64);

impl ModelRevision {
    /// Creates a revision from a raw value for cache adapters and tests.
    ///
    /// Model advancement is still owned by `Model`.
    #[must_use]
    pub const fn new(value: u64) -> Self {
        Self(value)
    }

    /// Returns the raw revision value for cache adapters, serialization, and test assertions.
    #[must_use]
    pub const fn get(self) -> u64 {
        self.0
    }

    pub(crate) fn advance(&mut self) {
        self.0 = self.0.checked_add(1).expect("model revision overflow");
    }
}
```
